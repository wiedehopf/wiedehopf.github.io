This is a minimal reproducer for a Chromium browser bug.

A variable is read as NaN despite not being NaN.

Update: setInterval context not necessary to reproduce
While this reproducer assigns testVar the value 0 * Math.PI,
i've observed this bug in a different context i can't reproduce where a simple
timestamp was assigned to the variable afterwards being read as NaN.

For this reproducer, using the multiplication of 0 and Math.PI or Math.E
seems necessary, i don't know why.

The function accessing testVar being in a different js source file was a
requirement to reproduce at least for me.

Open the development console to check if your browser is affected.
If you see "testVar is NaN" then the browser is affected.

Expected log for an unaffected browser will look like this:
"counters: anomaly: 0 correct: 10000 beforeAnomaly: 10000"

Expected log for an affected browser will look something like this:
"counters: anomaly: 2456 correct: 7544 beforeAnomaly: 10"
